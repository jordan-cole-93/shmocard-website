Badges — variants by role

Every badge uses .shm-badge + a role variant. Pills do four jobs: status, marketing tag, category, on-dark contrast.

Status / live.shm-badge--status + colour
Stock, availability, live state. Quiet pill (cream + hairline) with a coloured dot — the dot does the signalling, not the fill. Default to this for any “state” label.

Marketing tag.shm-badge--honey / --ember
Bestseller, sale, limited, save X%. Loud — used sparingly on product cards.

Category / filter.shm-badge default
Tags on cards, filter chips. Soft outline, no fill — recedes.

Value prop.shm-badge--ember / --cocoa + icon
Free shipping, fast setup — sits in product card or hero, paired with an icon.
Free shipping Fast setup
On dark surfaces--cream / --honey / --ember
Inside cocoa testimonial tiles or ember CTA sections. Default neutral disappears — flip up.
Available now Bestseller Limited

The rule. Pick the role first, then the variant.
• Quiet state → --status-* (dot signals, fill stays neutral).
• Marketing shout → --ember / --honey (full fill, max one per card).
• Category / filter → default .shm-badge.
• On dark surfaces → flip up to --cream / --honey / --ember.
Never style a state badge with a solid colour fill — a green pill reads as a marketing tag, not a status.
