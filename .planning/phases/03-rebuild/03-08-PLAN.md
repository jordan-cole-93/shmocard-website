**Cookie ↔ store reconciliation (RESEARCH.md Pitfall 6):**
- Cookie `shm-cart-id` is the source of truth for `cartId`
- Zustand persists `lines` (display) + UI state (`isOpen`)
- On first client mount: useCartHydration runs once, calls `getCartFromCookie()`, dispatches `setCart(cartId, checkoutUrl)` + replaces `lines`
- This prevents drift if Shopify expired or modified the cart

**Component tree (TRANSLATION.md cart drawer):**
```
<CartDrawer>                            // 'use client' — reads isOpen from useCart
  <CartScrim onClick={close} />         // backdrop
  <aside className="shm-cart" data-open={isOpen}>
    <CartHeader />                      // close button + title
    <CartFreeShipBand subtotal={...} /> // .shm-cart__ship; --unlocked variant when threshold hit
    {lines.length === 0
      ? <CartEmpty />
      : <>
          {lines.map(l => <CartLine key={l.id} line={l} />)}
          <CartSummary subtotal={...} total={...} />
          <CartCheckoutButton checkoutUrl={...} />
          <CartTrustStrip />
          <CartPaymentsStrip />
        </>}
  </aside>
</CartDrawer>
```
