```typescript

  'cr-80': 'google-reviews-nfc-tap-card-cr80',
  'l-sign': 'google-review-nfc-tap-card-l-sign',
  'square-card': 'google-review-plaque',
};
```
If URL slug not in map -> `notFound()`.

**Server-side fetch (from 03-12):**
```typescript
import { getProductByHandle } from '@/lib/shopify/queries';
const product = await getProductByHandle(HANDLE_MAP[urlSlug]);
if (!product) notFound();
```

**Slug-specific copy (lives in `pdp-copy.ts`):**
- `bullets[urlSlug]: string[]` — value props (CR-80: wallet size, hand-out per employee, etc.; L-Sign: counter-top; Square Card: adhesive)
- `programmingSteps: { n: number; title: string; body: string; }[]` — 4-5 steps describing the "we ship pre-loaded with Google review link" process; same for all formats
- `pdpFaqFor(slug): { q: string; a: string; }[]` — slug-specific PDP FAQ; common base + slug-specific items

**StickyBar implementation (RESEARCH.md anti-pattern warning):**
- It SLIDES DOWN from top on scroll past gallery — NEVER `position: fixed; bottom: 0`
- Uses `.shm-buybox-sticky` primitive from components.css
- Driver: `IntersectionObserver` on the gallery; toggle `data-visible="true"` on the sticky bar when gallery exits viewport
- Visual transitions handled by `.shm-buybox-sticky` CSS — JS only flips the data attribute

**Data discipline (REQ-06):**
- Product title, price, image URLs, variants, options come from `product.*` — NEVER hardcoded
- Programming process, returns, PDP FAQ, slug-specific bullets are MARKETING COPY — live in `pdp-copy.ts`
- Render `product.description` (plain text) — NEVER `descriptionHtml` (XSS surface; sanitizer out of scope)

**Pack selector pricing:**
The PDP shows pricing tiers from `product.variants[].price`. Variants represent pack sizes (e.g., 1-pack / 5-pack / 10-pack). PdpPackSelector displays each variant with its price; selection sets the active variant id; PdpQtyStepper qty multiplies by selected pack.
